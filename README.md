# ПрозорийЗвіт

Відкритий реєстр **прозорих благодійних зборів** для України: організатор публікує план витрат і етапні звіти, донор бачить статус довіри та може перевірити цілісність даних. Пілот у рамках [Web3 Resilience Lab](https://web3lab.org.ua).

**Платформа не приймає платежі** — лише посилання на зовнішні збори (Monobank тощо) та публічна звітність.

---

## Яку проблему вирішує

Після пожертви донор часто не має зручного способу відповісти на прості питання:

- Чи був **опублікований план**, куди підуть кошти?
- Чи з’явились **звіти з сумами** і підтвердженнями (чеки, акти)?
- Чи **витрати збігаються з планом**, а не лише з обіцянкою в пості?
- Чи організатор **регулярно звітує**, чи зник після збору?

Звіти розкидані по соцмережах, PDF у месенджерах, таблиці в Google — порівняти етапи і відстежити зміни важко. Довіра тримається на репутації, а не на перевірюваній структурі.

**ПрозорийЗвіт** зводить це в одне публічне місце: кампанія → план → звіти → журнал змін → криптографічний хеш звіту (готовність до фіксації в Web3).

---

## Що робить платформа

### Для донорів (без реєстрації)

- **Реєстр зборів** — усі публічні кампанії зі статусом довіри (в графіку, прострочено, завершено тощо).
- **Сторінка кампанії** — опис, зовнішнє посилання на збір, чеклист прозорості, порівняння **план vs факт**, історія звітів, audit log.
- **Каталог організаторів** — профіль ініціатора, контакти, історія зборів, агрегований індекс прозорості.
- **Перевірка хешу звіту** — SHA-256; донор може переконатися, що опублікований звіт не змінювали непомітно.
- **QR і публічне посилання** — зручно поширювати перевірений збір.

### Для організаторів (кабінет після входу)

- Створення кампанії, прив’язана до **профілю організації** (назва, ЄДРПОУ з профілю).
- **План витрат** за категоріями до або на старті збору.
- **Етапні звіти** з рядками витрат і вкладеннями (PDF, зображення).
- Автоматичний **журнал аудиту** при змінах кампанії, плану та звітів.

### Статуси довіри

Система підказує, наскільки збір «прозорий» за фактами: є план, є звіт, звітність вчасна, є документи, кампанія завершена зі звітністю — а не лише за красивим описом.

## Як це працює (логіка)

```
Організатор                    Публічна сторінка
     │                                │
     ├─ План витрат ────────────────► План vs факт
     ├─ Звіт + файли ───────────────► Звіт + SHA-256
     ├─ Зміна даних ────────────────► Audit log
     └─ Профіль організації ────────► /organizers/{slug}
```

Кошти збираються **зовні** (банківське посилання). Платформа фіксує **прозорість процесу**, а не проводить транзакції.

## Web3 та стійкість даних

- Кожен звіт отримує **SHA-256 хеш**; окрема сторінка верифікації для донорів.
- **Audit log** фіксує, що і коли змінилось у кампанії.
- Архітектура готова до **публічного anchor** хешу (блокчейн / IPFS) — наступний крок після MVP.

Без токенів, без збору криптовалют на платформі.
