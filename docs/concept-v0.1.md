# ПрозорийЗвіт v0.1

## Проблема

Донори не можуть швидко перевірити, чи організатор збору публікує структуровані звіти та чи витрати відповідають заявленому плану.

## Рішення

Відкритий реєстр кампаній: план витрат → етапні звіти → журнал змін → SHA-256 хеш звіту (готовність до публічного anchor).

## Цільова аудиторія

НКО, волонтерські збори, diaspora-донори, партнери програми Web3 Resilience Lab.

## Технології

Java 21, Spring Boot, PostgreSQL, Flyway, Thymeleaf. Без прийому платежів і без токенів.

## MVP (демо)

- Публічний реєстр і сторінка кампанії  
- Кабінет організатора: кампанія, план, звіт, файл  
- Статуси довіри (в т.ч. прострочення звіту)  
- Audit log
